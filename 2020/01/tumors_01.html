1. Introdução¶

Este tutorial inicia uma série de dois tutoriais que vão explorar dados diagnósticos de câncer de mama, criar e interpretar um modelo de aprendizado de máquina capaz de dicernir entre tumores de benignos e malignos.

Esse tutorial vai utilizar a linguagem Python e suas diversas bibliotecas para analisar os dados, então é esperado um conhecimento mínimo em Python e em estatística. Entretanto, ele foi escrito buscando explicar o máximo do código possível.

Aqui, os dados (disponibilizados abertamente pela UCI no Kaggle) serão explorados estatísticamente com o objetivo de extrair informações importantes sobre os tumores de câncer de mama.

2.1 Lidando com Imperfeições¶

Em dados do mundo real, como é o caso do que estamos trabalhando no presente tutorial, é quase que certeza que você vai encontrar diversos defeitos na sua base de dados.

Um dos defeitos mais comuns é a presença de colunas não preenchidas (no Python representadas por valores NaN). Esses podem não ser preenchidos por uma gama de motivos: descuido, impossibilidade de calcular/colher o dado daquele paciente ou até erro de manejo da base de dados.

De qualquer maneira, esses erros existem e é preciso saber identificar e lidar com eles. É possível identificar a presença de valores não preenchidos de maneira muito simples no Pandas:

2.3 Correlações, Inclusões e Exclusões¶

Para começar a entender como as nossas variáveis se relacionam como um todo, é imprescindível calcular a matriz de correlação. Essa matriz contêm o coeficiente de correlação entre cada uma das variáveis do nosso banco de dados.

O método aqui escolhido para calcular as correlações é o método Pearson, que calcula, para cada par de variável, um valor entre -1 e 1 representando a correlação entre esse par. Valores que se aproximam de 1 indicam uma correlação positiva forte, os perto de -1 indicam uma correlação negativa forte e os que se aproximam de 0 uma correlação fraca.

É preciso entender que o objetivo de uma matriz de correlação nesse estágio não é entender por completo que variáveis são significativas para montar um modelo complexo de aprendizado de máquina, mas sim para buscar variáveis interessantes e variáveis que provavelmente vão impactar negativamente o modelo, como vamos ver.

Para visualizar essa matriz, utilizaremos um heat map ou "mapa de calor", que será gerado nas linhas de código abaixo. Existem algumas funções que podem gerar esse mapa de maneira muito simples, como sns.heatmap do Seaborn, mas nenhuma delas produz, ao meu ver, um resultado agradável. Por isso, vou utilizar passar vários argumentos para a função sns.heatmap, com o objetivo de gerar um gráfico fácil de ler.

In [8]:
corr = df.corr(method='pearson') # Calcular a matriz de correlação pelo método Pearson

# Melhorar a estética do gráfico
mask = np.zeros_like(corr, dtype=np.bool)
mask[np.triu_indices_from(mask)] = True

plt.figure(figsize=(14, 11))

# Gerar o gráfico
sns.heatmap(corr, mask=mask, cmap='coolwarm', vmax=1.0, vmin=-1.0, center=0,
            square=True, linewidths=.5, cbar_kws={"shrink": .5})

2.4 Anomalias nos Dados¶

O último passo que tomaremos antes de visualizar alguns marcadores interessantes é identificar e tratar anomalias nos nossos dados. Porém, antes de buscar essas anomalias nós iremos normalizar o banco de dados para que comparações entre colunas diferentes fiquem mais fáceis de serem observadas.

A normalização que utilizaremos, 1-0 normalization), irá "encaixar" cada coluna no intervalo entre 0 e 1.

In [11]:
def normalize(df: pd.DataFrame) -> pd.DataFrame:
    """
    Normaliza o banco de dados entre (0..1).
    """
    
    norm_df = df.copy() # copiar o banco não-normalizado
    norm_df['Diagnóstico'] = norm_df['Diagnóstico'].apply(lambda diag: 1.0 if diag == 'M' else 0.0)

    norm_df = (norm_df - norm_df.min()) / (norm_df.max() - norm_df.min())
    norm_df['Diagnóstico'] = norm_df['Diagnóstico'].apply(lambda diag: 'Maligno' if diag == 1.0 else 'Benigno')

    return norm_df

Uma das anomalias mais presentes em bancos de dados, especialmentes aqueles preenchidos por pessoas, é a presença dos chamados outliers.

Outliers são pontos que destoam demais dos outros em sua distribuição e, por isso, acabam por prejudicar análises estatísticas por não representar pontos de dados reais, mas sim dados ruins ou mal calculados.

Uma maneira robusta de identificar outliers é visualizando o box plot dos dados. Idealmente, é interessante observar o box plot de todas as variáveis, mas, para poupar tempo, visualizaremos apenas os de quatro variáveis.

Em todas as variáveis analisadas, é possível observar uma presença significativa de outliers quando observamos os pontos desenhados acima do limite superior dos box plots. Com isso, é interessante remover parte desses pontos.

O box plot é calculado a partir de um intervalo da estatística descritiva denominado de IQR, que também utilizaremos para identificar e remover os outliers. Como não existe uma função que possa fazer tal tarefa nas bibliotecas que estamos utilizando, é necessário escrever uma manualmente:

In [13]:
def filter_iqr(df, col_name):
    col = df[[col_name]]
    
    Q1 = col.quantile(.25) # intervalo onde se compreende 25% do banco de dados
    Q3 = col.quantile(.75) # intervalo onde se compreende 75% do banco de dados

    IQR = Q3 - Q1 # diferença entre esses dois intervalos: o "meio do banco de dados"
    
    # remover as pessoas que estão fora da margem do intervalo IQR com 1.5 de margem
    col = col[~((col < (Q1 - 1.5 * IQR)) |(col > (Q3 + 1.5 * IQR))).any(axis=1)]
    
    # aplicar os resultados
    result = df.copy()
    result[col_name] = col

    return result.dropna()

A primeira diferença notável é que o tamanho do banco de dados caiu de 569 para 487, tornando-se aproximadamente 14% menor. Isso pode ser visto como um problema, pois perdemos uma quantidade considerável de informação ao remover os outliers.

A grande questão é: essa informação perdida é relevante? Ela é uma informação construtiva ou destrutiva?

Essas perguntas só podem ser devidamente respondidas com experimentações e testes que não serão cobertos nesse simples tutorial. De qualquer forma, problemas de trade off como esse são comuns na ciência de dados, então é interessante se acostumar com a presença deles.

Se plotarmos o mesmo gráfico plotado acima, deveremos observar outliers de magnitude muito menor. Isto é, enquanto antes nós tinhamos outliers em grande quantidade e bem espaçados, aqui esperamos observar outliers mais compactos e até inexistentes em uma das variáveis.

É importante notar que, por conta de nós normalizarmos o banco de dados imediatamente antes de desenhar cada box plot, não é possível comparar a magnitude dos valores entre gráficos pois a normalização sempre vai "encaixar" os valores entre 0 e 1.

sns.boxplot(data=normalize(df)[['Raio Médio', 'Textura', 'Concavidade Média', 'Suavidade Média', 'Compactez Média']])

3. Visualizando Marcadores¶

Agora que nós usamos a análise exploratória para encontrar que características eliminar, é importante entender que variáves poderão ser utilizadas pelo modelo para diferenciar entre tumores malignos e benignos.

O violin plot - "gráfico de violino", no português - ajuda a visualizar a probabilidade de uma variável ser útil ou não para diferenciar entre duas classes. Por sorte, a biblioteca Seaborn consegue gerar esses gráficos.

Vamos definir que variáveis queremos observar:

In [16]:
variaveis = ['Diagnóstico', 'Raio Médio', 'Textura', 
             'Concavidade Média', 'Suavidade Média', 'Compactez Média']

Agora usaremos um pequeno truque (que usa uma função um tanto complicada do Pandas, melt) para conseguir desenhar o gráfico de várias variáveis diferentes lado a lado, permitindo não somente visualizar o poder de discriminação de uma delas, mas a comparação deste entre variáveis.

In [17]:
melted_df = normalize(df)[variaveis].melt(id_vars='Diagnóstico',var_name='Variável', value_name='Valor')

sns.violinplot(x="Variável", y="Valor", hue='Diagnóstico', split=True, inner="quart", data=melted_df)
Out[17]:
<matplotlib.axes._subplots.AxesSubplot at 0x7fa7d7655da0>

Para entender o que esses gráficos bonitos querem nos dizer, é preciso entender o que as curvas azuis e laranjas significam.

Cada curva, azul ou laranja, representa a densidade probabilística daquela variável, isto é, representa a distribuição da variável em termos probabilísticos - são curvas KDE. As curvas estão organizadas em pares, onde a esquerda (azul) representa a distribuição em tumores malignos e direita (laranja) em tumores benignos.

Por exemplo, a distribuição em tumores malignos do raio médio é muito mais concentrada em valores maiores, enquanto nos benignos essa distribuição é consideravelmente mais concentrada em valores menores. Logo, a variável "Raio Médio" provavelmente será útil para discriminar entre esses dois tipos de tumores.

O mesmo acontece em "Textura", "Concavidade Média" e "Compactez Média". É possível visualizar certa discriminação em "Suavidade Média" também, mas ela é bem menos clara que as outras.
